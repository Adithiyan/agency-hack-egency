    How Phantom Flow works

    A step-by-step guide to how public grant data becomes an enforcement triage queue.

    1. 1
      Ingest federal grants data

      Downloads the Government of Canada proactive disclosure dataset (~200K+ rows). Filters to for-profit, nonprofits, and Indigenous recipients with awards ≥ $25,000. Normalizes entity names — strips legal suffixes, collapses punctuation.

      open.canada.ca/data recipient_type: F, N, A, O
    2. 2
      Aggregate by recipient

      Groups all awards by normalized entity name. Builds a profile per entity: total awarded, number of grants, first/last award date, funding years, departments, and programs. Derives funding concentration (years × departments) as a proxy for public revenue dependency.

    3. 3
      Match to Corporations Canada

      Looks up each entity against the federal corporate registry. Uses fuzzy token-sort matching with a confidence score. Retrieves corporate status, dissolution date, and jurisdiction. Low-confidence matches are flagged separately to avoid false positives.

      High ≥ 92 Medium 80–91 Low 65–79
    4. 4
      Flag zombies (12-month window)

      Marks an entity as a zombie if its corporate status is Dissolved, Bankrupt, Inactive, Cancelled, or Struck — AND the dissolution date falls within 12 months of the last grant award. This aligns exactly with the challenge definition. An extended 24-month view is flagged separately.

      zombie_12mo zombie_extended
    5. 5
      Score recovery ROI (4 components)
      ComponentWeightLogic
      Recoverable amount35Award size × (1 − months/12)
      Evidence strength30Match confidence + dissolution date present
      Pursuit cost20Penalises weak matches + old cases
      Public exposure15Funding dependency level + amount scale
    6. 6
      AI case summaries (Gemini / Claude / Groq)

      Generates a 3-sentence enforcement triage summary for each top case. The model receives only structured facts — no free-text retrieval. It cannot invent numbers, dates, or names. A deterministic template is used when no API key is configured.

    7. ★
      AI autonomous investigation

      The "Let the AI investigate" button sends the top 8 zombie candidates to the LLM. The model autonomously picks the single most compelling case, explains its reasoning, writes a 200-word CBC-style forensic narrative, and lists specific red flags — each citing data from the evidence. The model decides; no case is pre-coded as the answer.

    Known limitations

    • No total revenue data — funding dependency is a proxy (years × departments), not a measured ratio.
    • Federal corporate registry only — misses provincial registrations (BC, ON, QC).
    • Name matching can produce false positives; low-confidence cases are routed to review, not enforcement.
    • Demo data uses 20 representative entities. Live pipeline downloads the full 200K-row dataset.

    Settings & API Keys

    What API keys do you need?

    LLM API key — for AI case summaries. Supports Gemini (ai.google.dev), Claude (console.anthropic.com), or Groq (console.groq.com). Set PHANTOM_FLOW_LLM_PROVIDER in .env.
    Grants CSV — no key needed. Public download from open.canada.ca (Upload tab).
    Corporations Canada — no key needed for basic search. Demo uses offline fixture.
